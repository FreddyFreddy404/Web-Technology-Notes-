1. Basics, Scope & Operators

let, const & var

var: Can be re-declared & updated. Global scope. (Avoid using this in modern JS).

let: Cannot be re-declared, but CAN be updated. Block scope.

const: Cannot be re-declared OR updated. Block scope.

Code

const a = 10;
// a = 20; // ERROR!

{ // Block 1
  const x = 5;
}
{ // Block 2
  const x = 15; // Valid! Different block.
}

Equality: == vs ===

== only checks the value (it converts types automatically).
=== (Strict Equality) checks both value AND type. Always use ===!

let a = 5;
let b = "5";
console.log(a == b); // Value match
console.log(a === b); // Type mismatch

Console Output

true
false

Interaction & Comments

console.log("..."): Prints to terminal/console.
alert("..."): Webpage popup warning.
prompt("..."): Webpage popup that takes input (returns a string).

Comments: // Single line or /* Multi line */

2. Objects & Strings

What is an Object?

An object is a container that stores related data + functions together.

Why Objects Matter in JS: Everything important in JS uses objects (buttons, arrays, dates, users, games, APIs, forms). Even arrays are objects!

let zombie = {
  health: 20,
  name: "Zombie",
  isBaby: false,
  attack: function() {
    console.log("Zombie attacks!");
  }
};

// Use it:
console.log(zombie.health);
zombie.attack();

Console Output

20
Zombie attacks!

Objects (const Mutation)

Even if an object is declared with const, you can still modify its internal keys! You just can't reassign the entire object variable to something else.

const student = {
  name: "Freddy",
  age: 19
};
// Valid: Changing a key's value
student.name = "Marmik";
student["age"] = 20;

Template Literals (String Interpolation)

Use backticks ` ` instead of quotes. Inject variables directly using ${variable}. (Highly asked in interviews!)

let item = "pen";
let price = 10;
let out = `The cost of ${item} is ${price} rs.`;
console.log(out);

Console Output

The cost of pen is 10 rs.

3. Loops & Iteration

Standard for, while, and do...while are identical to C/Java. JS adds two special loops:

for...of (Arrays & Strings)

Iterates over the values of iterable objects (like arrays or strings). No manual incrementation needed.

let str = "Hi";
for (let i of str) {
  console.log(i);
}

Console Output

H
i

for...in (Objects)

Iterates over the keys of an object.

const obj = { a: 1, b: 2 };
for (let key in obj) {
  console.log(`key=${key}, val=${obj[key]}`);
}

Console Output

key=a, val=1
key=b, val=2

4. Functions & Array Methods

Functions & Arrow Functions

Higher Order Functions: Functions that take other functions as parameters (callbacks) or return them.

Arrow Functions () => {}: A modern, shorter way to write functions. Used heavily in React.

// Normal Function
function sum(a, b) {
  return a + b;
}

// Arrow Function equivalent
const arrowSum = (a, b) => {
  return a + b;
};

Array: forEach()

Used strictly to execute a function on every item in an array. It does NOT return a new array.
Parameters: (value, index, array)

let fruits = ["apple", "banana"];
fruits.forEach((fruit, idx) => {
  console.log(idx, fruit);
});

Console Output

0 'apple'
1 'banana'

Array: map(), filter(), reduce()

Unlike forEach, map() returns a brand new array based on the operation. No need to cast it to a list like in Python!

let nums = [1, 2, 3];
// MAP: Modifies each element
let doubled = nums.map(n => n * 2);

// FILTER: Keeps elements that return true
let evens = nums.filter(n => n % 2 === 0);

// REDUCE: Boils array down to single value (e.g. sum)
let sum = nums.reduce((res, curr) => res + curr);

5. DOM Manipulation (Detailed)

DOM (Document Object Model): The browser's tree representation of HTML. The window is the global browser object; document is the HTML page inside it. Use console.dir(document) to inspect it.

Selecting Elements

document.getElementById("myId"): Returns 1 element.
document.getElementsByClassName("myClass"): Returns HTMLCollection.
document.querySelector(".myClass"): Returns the FIRST matching element (Can take tags, #ids, .classes).
document.querySelectorAll("p"): Returns a NodeList of ALL matching elements.

innerText vs innerHTML vs textContent

PropertyWhat it Returns / Sets
innerTextVisible text only (ignores hidden elements)
innerHTMLFull HTML tags + text. (Can be used to inject tags)
textContentAll text, even if hidden by CSS display:none
// HTML: <h1>Hi</h1>
let h1 = document.querySelector("h1");
h1.innerText += " Freddy"; // Appends text
h1.innerHTML = "<i>Italic Freddy</i>"; // Injects HTML

Creating & Deleting Elements

1. Create: let el = document.createElement("div")
2. Append (Inside at end): node.append(el)
3. Prepend (Inside at start): node.prepend(el)
4. Before (Outside, above): node.before(el)
5. After (Outside, below): node.after(el)
6. Delete: node.remove()

// Targetting: <div id="box">Middle Text</div>
let box = document.querySelector("#box");

let btn1 = document.createElement("button");
btn1.innerText = "Prepended";
box.prepend(btn1);

let btn2 = document.createElement("button");
btn2.innerText = "Appended";
box.append(btn2);

let para = document.createElement("p");
para.innerText = "Before (Outside)";
box.before(para);

// box.remove(); // Removes the entire box

6. Events

Note: JS Event Handling has priority over inline HTML event handling.

addEventListener & Event Object (e)

addEventListener = Wait for something, then run function.
e = Info about what happened.

let btn = document.querySelector("#myBtn");

const greet = (e) => {
  console.log("Event type:", e.type);
};

btn.addEventListener("click", greet);

What this means:

btn.addEventListener("click", greet);

“When button is clicked, run greet and give event details.”

7. Classes & Object-Oriented Programming (OOP)

Line-by-Line Breakdown

Super Easy Meaning:
Person = Human
Engineer = Human + coding skill
Freddy = one Engineer

Most Important Line: class Engineer extends Person = Engineer gets Person powers.

class Person {
  constructor(name) {
    this.species = "Homo Sapiens";
    this.name = name;
  }
  eat() { console.log("eating"); }
}

class Engineer extends Person {
  constructor(name, branch) {
    super(name);
    this.branch = branch;
  }
  work() { console.log("coding"); }
}

let freddy = new Engineer("Freddy", "CS");
freddy.work();
freddy.eat();

What happens step-by-step?

Creates a class. Think: blueprint for making humans.

constructor(name) {
Special function that runs automatically when new object is made.

this.species = "Homo Sapiens";
Adds property to object. this = current object.

class Engineer extends Person
Engineer gets all features of Person.

super(name);
Runs parent (Person) constructor. Adds species and name.

let freddy = new Engineer("Freddy", "CS");
Creates object. Values become: name=Freddy, branch=CS, species=Homo Sapiens.

freddy.work();
Runs Engineer's method. Output: coding

freddy.eat();

8. Async JS, Promises & Fetch API (Completion)

JavaScript runs Synchronously (one line at a time). However, tasks like fetching data from a server take time. If JS waits, the website freezes. Therefore, we use Asynchronous programming.

setTimeout & Callbacks

setTimeout(callback, milliseconds) executes code AFTER a timer. This is Async! JS doesn't stop and wait; it continues running other code and comes back to this when the timer finishes.

console.log("1");
setTimeout(() => {
  console.log("2"); // Runs after 2 seconds
}, 2000);
console.log("3");

Console Output

1
3
2

Promises (Solving Callback Hell)

When you have nested callbacks (callbacks inside callbacks inside callbacks), it becomes "Callback Hell" (unreadable code). Promises solve this.
A Promise is an object representing eventual completion (or failure) of an async task. It has 3 states: Pending, Resolved (success), Rejected (error).

let myPromise = new Promise((resolve, reject) => {
  let success = true;
  if(success) resolve("Data Fetched!");
  else reject("Network Error");
});

// Consuming the promise:
myPromise.then((res) => {
  console.log(res); // Runs if resolved
}).catch((err) => {
  console.log(err); // Runs if rejected
});

Async / Await

An even better, cleaner way to write Promises! Putting async before a function makes it always return a Promise. Inside it, using await pauses execution until the promise settles, making async code look synchronous!

async function getData() {
  console.log("Fetching data...");
  await setTimeout(...some DB query...);
  console.log("Done!");
}

Fetch API (Network Requests)

Used to get data from other websites/servers. fetch() returns a Promise. You must resolve it, and then convert the result to JSON (which also returns a Promise).

const URL = "https://api.github.com/users/marmik";

async function getFacts() {
  // 1. Await the network request
  let response = await fetch(URL);
  console.log(response.status); // e.g., 200

  // 2. Await the JSON conversion
  let data = await response.json();
  console.log(data);
}

getFacts();
