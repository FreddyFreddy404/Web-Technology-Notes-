1. Introduction & Basics

HTML Definition

HyperText Markup Language

- Not a programming language, but a markup language.

- Its single main job is to describe the *structure* of a web page.

What Happens on google.com

Typing google.com triggers:
1. DNS lookup (finds Google's IP address).
2. HTTP request from your browser to the server.
3. Server response (sends HTML, CSS, JS files).
4. Browser renders (reads HTML top-to-bottom and displays page).

Syntax, Case Sensitivity & Comments

Tags <>: Elements are enclosed in opening <tag> and closing </tag> pairs.

Attributes: Add info *inside* the opening tag (e.g., id="main"), as name/value pairs.

Case Sensitivity: Not technically case sensitive, so <hEAd>...</HeAd> works. *Best practice is always lowercase!*

Comments: Use <!-- comment -->. SEO bots *completely ignore* them!

Why index.html?

Web servers automatically look for index.html as the default page for a directory. Other names require typing the full filename in the URL.

2. Document Structure

Core Skeleton Tags

<!DOCTYPE html>: Mandates HTML5 standards usage.

<html lang="en">: Root element; lang="en" is vital for accessibility (screen readers) and SEO (specifying language).

<head>: Meta-info, links (CSS), title – *none* are visible on the page.

<title>: Browser tab text; crucial for SEO and user recognition.

<body>: *All* visible content goes here.

Code Skeleton

<!DOCTYPE html>
<html lang="en">
<head>
    <title>My Page</title>
</head>
<body>
    Visible content goes here.
</body>
</html>

3. Text Formatting Elements

Always use appropriate tags! Randomly typed text (e.g., just "hello" in the body) works but lacks structure, meaning, and consistent styling across browsers.

<h1> to <h6> Block Element

Headings by size/importance. *Don't use for sizing; use <h1> only for the *main* page topic for strong SEO.*

Heading Code

<h1>Main Title</h1>
<h3>Sub Title</h3>
<h6>Smallest Title</h6>

Visual Output

Main Title

Sub Title

Smallest Title

<p> , <pre> & Formatting tags

<p> (Block): Paragraph tag; standard text block.

<pre> (Block): Preformatted text; preserves all whitespace and line breaks exactly.

Inline Formatters: Use <strong> (bold + semantic meaning), <b> (only bold), <i> (italic), <u> (underline), <sub> (subscript), <sup> (superscript).

Note: <big> and <small> are largely deprecated; use CSS for size control!

Formatting Code

<p>A normal paragraph with <strong>important</strong> text and <u>underlined</u> text.</p>
<p>H<sub>2</sub>O and E=mc<sup>2</sup>.</p>
<pre>
  This   keeps
    weird
  spacing!
</pre>

<hr> & Self-Closing Tags Block Element

Injects a horizontal thematic break (line). No content inside, so no closing tag needed!

Common Self-Closers: <hr>, <br> (line break), <img>, <input>, <meta>.

HR Code

<p>Text above</p>
<hr>
<p>Text below</p>

4. Links & Media

<a> (Anchor Tag) Inline Element

Creates links. Attributes: href="..." (destination URL). No text = nothing to click; no href = unclickable text.

Absolute vs Relative: Absolute links are full URLs (https://google.com); relative links point to local files (e.g., /about.html).

Open in new tab: Add target="_blank".

Anchor Code

<a href="https://wikipedia.com" target="_blank">Wikipedia (New Tab)</a>

Visual Output

Wikipedia (New Tab)

<img> and Image Links Inline Element

Displays images. Attributes: src (path), alt (alternative text – crucial for accessibility/failed loads). *Add loading="lazy" for performance (loads image *only* when visible)!*

Clickable Image: Wrap the <img> inside an <a> tag.

Image Code

<a href="https://google.com" target="_blank">
  <img src="https://www.google.com/favicon.ico" alt="Google Logo" loading="lazy">
</a>

Visual Output

<iframe> & <video> Block / Inline-Block

<iframe>: Embeds *another* HTML doc (like YouTube/maps).

<video>: Embeds local video. Attributes: controls (player buttons), autoplay, loop, muted.

Video Code

<video width="250" controls muted>
  <source src="movie.mp4" type="video/mp4">
</video>

5. Semantics & Layout

Block Elements span full width, new line. Inline Elements span necessary width, no new line. Non-semantic tags (div/span) are for structure *without* meaning, while semantic tags describe meaning clearly.

Non-Semantic Tags: <div> & <span>

Tell browsers *nothing* about content, purely for grouping/CSS styling.

<div> (Block): Generic container for block-level content; most common layout element.

<span> (Inline): Generic container for inline content (styling a single word).

Div/Span Code

<div style="background-color: #3a3a3a; padding: 15px; border-radius:5px;">
  A block. Highlight a <span style="color: #ff5e5e; font-weight: bold;">single word</span>.
</div>

Visual Output

A block. Highlight a single word.

Semantic Tags Block Elements

Clear meaning to browser/developer. Pros: **(1) Readability**, **(2) SEO** optimization, **(3) Accessibility** (screen readers navigate better).

<header>: Page/section intro, nav; often has <h1>, logos, search.

<nav>: specifically for *major* navigation links.

<main>: Unique, dominant body content (can hold <article>, <section>, <aside>).

<footer>: Page bottom; copyright, contact, sitemaps.

6. Lists

<ul>, <ol>, & <li> Block Elements

<ul>: Unordered (Bullets).

<ol>: Ordered (Numbers). Custom type attribute (e.g., type="A" for letters, type="I" for Roman numerals).

<li>: List Item (inside *both* UL/OL).

7. Tables

Table Structure & Colspan

Display grid data only – never use for layouts!

<thead> / <tbody>: Semantically group header/body rows.

<tr>: Row. <th>: Header cell (bold, centered). <td>: Data cell.

<caption>: Table title.

colspan="...": Attribute to stretch cell across columns (merge columns).

Table Code

<table class="demo-table">
  <caption>Monthly Savings</caption>
  <thead>
    <tr><th>Month</th><th>Amount</th></tr>
  </thead>
  <tbody>
    <tr><td>Jan</td><td>$100</td></tr>
    <tr><td colspan="2" style="text-align:center; background-color:#2a2a2a; color:#fff;">No Feb savings</td></tr>
  </tbody>
</table>

8. Forms

Form Basics, Inputs, & Attributes

<form>: Input wrapper; action="..." defines data destination.

ID vs Class: ID is unique (one per page); class is a group identifier (multiple elements share).

<label for="...">: Text for input; for must match input's ID (clicking label hits input!).

Attributes: placeholder (hint text), name (server variable name), value (default text).

<textarea>: Multi-line, variable-sized text input.

<select> & <option>: Dropdown menu.

Form Code

<form class="demo-form" onsubmit="event.preventDefault();">
  <label for="demouser">Name:</label>
  <input type="text" id="demouser" placeholder="Enter name">
  
  <label>Role:</label>
  <select>
    <option>Admin</option>
    <option>User</option>
  </select>
  
  <label>Bio:</label>
  <textarea rows="3"></textarea>
  
  <input type="submit" value="Register">
</form>

Visual Output
